# Analysis: v1 Initial Benchmark — Per-Projection Fused LoRA Matmul

**Date**: 2026-05-24
**Kernel version**: v1
**GPU**: NVIDIA A100-SXM4-80GB
**Compared against**: PyTorch reference (matmul_lora), cuBLAS bare matmul

## Summary

v1 fused LoRA matmul is **0.70x Unsloth** and **0.78x PyTorch** on full QKV at rank=16. Unsloth's `addmm_` pattern is notably faster than naive PyTorch (Unsloth 2.033ms vs PyTorch 2.250ms). Our Triton matmul achieves ~0.57-0.68x cuBLAS, matching the lora_mlp v1 experience. The fundamental bottleneck is the base Triton matmul, not the LoRA fusion.

### Key observation: Unsloth is faster than naive PyTorch

Unsloth's `addmm_` pattern saves ~10% over naive PyTorch (`out + s * (XA @ B.t())`):
- Q-projection rank=16: PyTorch 1.396ms vs Unsloth 1.226ms (12% faster)
- Full QKV rank=16: PyTorch 2.250ms vs Unsloth 2.033ms (10% faster)

This means our target for v2+ is Unsloth, not PyTorch. Beating PyTorch is easy; beating Unsloth requires matching cuBLAS matmul speed.

## Bottleneck Identification

### Primary bottleneck: Triton base matmul is ~0.58-0.69x cuBLAS

The Triton tiled matmul achieves 58-69% of cuBLAS throughput on the Q-projection shape. This is the same fundamental issue seen in lora_mlp v1 (~73% cuBLAS). Despite adding L2 swizzle, we don't match cuBLAS because:

1. **cuBLAS has architecture-tuned kernels**: precomputed optimal tile configs, warp scheduling, software pipelining depth
2. **Triton autotune explores limited configs**: we have 10 configs; cuBLAS has hundreds
3. **The LoRA post-pass reads X a second time**: adds ~30% overhead at rank 8, growing to ~70% at rank 64

### Secondary bottleneck: LoRA overhead scales with rank

| Rank | v1 base (no LoRA, estimated) | v1 with LoRA | LoRA overhead |
|------|------|------|------|
| 8  | ~1.6ms | 1.831ms | ~14% |
| 16 | ~1.6ms | 1.853ms | ~16% |
| 32 | ~1.6ms | 1.885ms | ~18% |
| 64 | ~1.6ms | 2.261ms | ~41% |

The LoRA post-pass does a second K-loop over X × A (r columns), then one small tl.dot for XA × B. At rank 64 the XA accumulator is [128, 64] = 32KB fp32, creating register pressure.

### K/V projections (GQA) are relatively better

At N=1024 (K/V), our kernel is 0.89x PyTorch at rank 8 — closer to parity because the smaller N dimension means less total work and the LoRA overhead is proportionally smaller.

## Recommended Next Steps

1. **v1_2: Fuse the LoRA K-loop into the base K-loop** (like lora_mlp v1 did) — trades register pressure for eliminating the second X read. May help at low ranks but hurt at high ranks.

2. **v2: cuBLAS + Triton epilogue approach** (applying lora_mlp v3 lesson directly):
   - cuBLAS for X @ W^T (base matmul — full cuBLAS speed)
   - cuBLAS for X @ A^T (skinny GEMM — cuBLAS handles this in ~0.07ms)
   - Single Triton kernel: read base output, add s * XA @ B^T, write final result
   - This should achieve ~1.0x+ PyTorch since the matmul is at full cuBLAS speed

3. **v3: Cross-projection fusion** — load X once, compute all 3 outputs. This is where QKV differs most from MLP (no activation barrier).

## Cross-Version Comparison (Full QKV, rank=16, bf16, A100)

| Kernel | Approach | Time (ms) | vs Unsloth | vs PyTorch | Launches | X HBM reads |
|--------|----------|-----------|------------|------------|----------|-------------|
| cuBLAS bare (Q+K+V, no LoRA) | 3× torch.matmul | ~1.7ms | — | — | 3 | 3× |
| **PyTorch naive** | 3× (matmul + XA + add) | 2.250ms | 0.90x | 1.00x | 9+ | 6× |
| **Unsloth** | 3× (matmul + XA + addmm\_) | **2.033ms** | **1.00x** | 1.11x | 9 | 6× |
| **Triton v1** | 3× fused Triton kernel | 2.895ms | 0.70x | 0.78x | 3 | 6× (2 per proj) |

**Bottom line**: Unsloth is the bar to beat. It's 11% faster than naive PyTorch thanks to `addmm_`. Our v1 is 30% slower than Unsloth because the Triton base matmul can't match cuBLAS.

**Path to v2**: Use cuBLAS for all matmuls (matching Unsloth's base speed), then add a Triton epilogue kernel to fuse the LoRA addition across Q/K/V — saving the 3 HBM round-trips for X@A intermediates and reducing X reads from 6 to 3 (or 1 if we also fuse the A matmuls).
